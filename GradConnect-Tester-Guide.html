Getting set up

Before you start

GradConnect runs locally through Laravel Herd. Open the address your team lead shares — when running on the host machine that is http://gradconnect.test.

Two ways the AI features behave

The résumé & job embeddings and fit scoring run as background jobs. There are two states to be aware of when testing:

  • The alumni@gradconnect.edu.ph account already has a résumé and a full set of AI recommendations loaded — you can view those immediately, no setup needed.
  • To generate new matches (uploading a fresh résumé, or clicking Refresh recommendations / Refresh matches), the host must have a queue worker running and AI keys configured.
For the person hosting the app Run the queue worker so résumé uploads and the refresh buttons process live:
php artisan queue:work
The Gemini and Groq API keys must be set in .env (GEMINI_API_KEY, GROQ_API_KEY). Without them, uploads still save but stay unscored.

Alumni

A graduate managing their career profile, résumés, job applications, and AI recommendations. This is the best account to see the AI matching in action.

What you can do

  • My Profile — fill in career details across four tabs: Basic Info, Education, Employment, Skills.
  • My Résumés — upload a PDF résumé (this is what gets matched to jobs).
  • Job Board — browse open positions, filter, and apply.
  • Recommendations — see jobs ranked by AI fit score with explanations and skill gaps.
  • Surveys — answer open employability & tracer surveys.

Try the AI matching

  • Open Recommendations — alumni@ already shows 6 ranked jobs (Backend Developer and Software Engineer score highest for its backend résumé).
  • Each card shows a fit bar, the model's explanation, and green "matched" / amber "gap" skill chips.
  • Upload your own résumé under My Résumés (see the samples below), then press Refresh recommendations to generate a fresh ranking.
Résumés must be PDF(max 10 MB). Other formats are rejected. New uploads only get scored while the host's queue worker is running.

Set expectations

Known gaps — not bugs

This build covers the foundation plus the AI matching feature. A few menu items are intentional placeholders for future work — please don't log these as defects:

  • Sidebar links that do nothing yet: Career Resources, Talent Search, Job Moderation, System Reports, Settings, Alumni Records, Career Activities, Graduate Profiles.
  • The Employability Report is a basic summary (totals and status breakdown), not a full analytics suite.
  • Dashboard numbers reflect current data — AI match counts climb only after résumés are uploaded and scored.
  • Self-registration creates an account with no role; an Admin assigns roles afterward. For testing, use the seeded logins above.

Found a real issue? Note the role, the page, and what you expected vs. saw — that's enough to reproduce it quickly.
